## Reusable Logic

- ERPGo is the best ERP reference: accounting, invoices, expenses, payments, suppliers, HRM/CRM/POS, document events, package modules. Salesy adds CRM/quotes/opportunities. UltimatePOS adds cash register and account transactions but is `nulled`.
- Recommended B2B schema: `erp_document_sequences`, `erp_suppliers`, `erp_purchase_orders`, `erp_purchase_invoices`, `erp_expenses`, `erp_payment_allocations`, `erp_accounts`, `erp_ledger_entries`, `erp_tax_rates`, `erp_approval_requests`, `erp_branches`, `erp_vendor_ledgers`.

## NeoGiga Adaptation Plan

1. Start with document numbering and audit trail before accounting.
2. Adapt ERPGo document/event patterns into NeoGiga services: `DocumentNumberService`, `ApprovalWorkflowService`, `LedgerPostingService`.
3. Keep marketplace orders as commerce source, then post summarized ledger entries into ERP tables.
4. Add supplier purchase flow tied to inventory receiving.
5. Implement role-scoped admin APIs and tests for balanced ledger posting.

## Gaps / Risks

- ERPGo includes SaaS/multi-company assumptions; adapt to NeoGiga marketplace/vendor/warehouse boundaries.
- Avoid copying payment integrations or billing code before security review.
