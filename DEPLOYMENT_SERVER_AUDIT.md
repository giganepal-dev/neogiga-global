## Conflicts between the deployment spec and the live environment
1. **DB engine:** spec mandates **MySQL `neogiga_prod`** (utf8mb4, mysqldump, :3306); live NeoGiga runs on **PostgreSQL `neogiga`** (working, seeded, Blueprint-mandated, built on prior instruction).
2. **Directory:** spec wants `/var/www/neogiga/{backend,admin,frontend,...}`; live is `/home/neogiga/laravel` under Virtualmin. `/var/www/neogiga` does not exist.
3. **Web user:** spec says `chown www-data:www-data`; this box runs the site as user `neogiga` (Virtualmin suexec). Using www-data would break isolation.
4. **Web server management:** spec describes hand-written Nginx/Apache vhosts; here Apache vhosts are **Virtualmin-managed** and can be regenerated by Virtualmin — hand edits must go in include files or be made Virtualmin-aware.

These require a decision before Phase 3+ (see NEOGIGA_LIVE_DEPLOYMENT_REPORT.md).
