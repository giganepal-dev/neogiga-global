# Product and Category SEO Pattern Audit

Date: 2026-07-14

## Existing state

- Product/category SEO is generated by `CatalogSeoContentGenerator`, but its title/description patterns do not match the newly approved Global/Nepal/regional wording.
- Many generated product canonicals were persisted using the execution host (including localhost) rather than the configured marketplace origin and complete localized route.
- Product detail reads persisted values directly, so incorrect generated values can override correct marketplace runtime context.
- Category detail reads its `seo_meta` JSON directly and does not generate marketplace-specific active metadata.
- `MarketplaceSeoRenderer` correctly appends the current path to a configured origin, but entity-level title/description/indexability is not centralized.
- Product SEO rows do not separate generated values from manual active values, lock state, reasoned robots decisions, or version history.
- The sitemap emits legacy plural brand URLs and does not use the same centralized entity eligibility rules as pages.
- Hreflang is generated centrally and can be retained, but entity paths and cache identity must remain host/locale aware.

## Integration decision

Add one marketplace-aware catalog SEO template service and an indexability service. Runtime pages use the approved pattern for the current marketplace while a bulk command persists Global generated values only. Existing admin/manual or locked values remain active; generated fields, active source, template version, reason, modifier, timestamps, and append-only version snapshots are tracked separately.

The centralized service will use configured `seo_marketplace_name`, `has_local_warehouse`, `warehouse_display_name`, `seo_fulfilment_phrase`, and `seo_site_suffix` fields with conservative fallbacks. It will never claim a country warehouse unless configuration explicitly allows it.

## Implemented outcome

- `CatalogSeoTemplateService` now owns the approved product, category, brand and manufacturer patterns, complete localized canonical paths and reasoned indexability decisions.
- Product SEO separates generated values from active manual values and adds lock/source/template/reason/modifier/timestamp fields. Category JSON follows the same governance contract.
- `catalog_seo_versions` records append-only product/category snapshots with source notes, confidence, last-updated time and the required “Advisory only” disclaimer. Product admin exposes history and guarded rollback.
- `seo:catalog-regenerate` is dry-run-first, chunked/resumable, generated-only and manual/lock preserving. Production processed 73,058 products and 441 categories: 69,879 products and 179 categories changed; 3,179 product and 262 category records were preserved. The follow-up dry run reported zero changes.
- Production now has 73,058 product SEO rows and 70,058 new version snapshots. Canonical product/category/brand URLs, hreflang and sitemap shards were verified after deployment.
